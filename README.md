# MMR Lens

라이엇이 공개하지 않는 롤 솔로랭크의 숨겨진 MMR을 추정해 보여주는 웹사이트.

최근 솔로랭크 경기에서 같은 로비에 배정된 플레이어들의 현재 랭크를 역추적해,
"실제로 어느 실력대와 매칭되고 있는지"를 계산합니다.

## 스택

- **Next.js 16** (App Router, React 19)
- **shadcn/ui** + Tailwind CSS v4 (다크모드 지원)
- **Recharts** (shadcn chart) — MMR 추이 그래프
- **캐시**: 기본 인메모리 / `DATABASE_URL` 설정 시 Postgres(Supabase) 자동 사용

## 실행

```bash
npm install
cp .env.example .env.local  # RIOT_API_KEY 채우기
npm run dev
```

> 라이엇 개발용 API 키는 https://developer.riotgames.com 에서 발급하며 **24시간마다 만료**됩니다.

## MMR 추정 방식

1. 최근 솔로랭크 10경기를 조회해 리메이크(5분 미만)를 제외하고 8경기 분석
   - 같은 팀에 2회 이상 등장한 플레이어가 있는 경기는 **듀오로 간주해 분석 제외**
     (듀오 로비는 파트너 MMR에 영향을 받아 추정을 오염시킴 · 제외 후 4경기 미만이면 제외 포기)
2. 경기마다 양 팀에서 3명씩 샘플링해 참가자들의 **현재 솔로랭크**를 조회
3. 티어/디비전/LP를 단일 포인트로 환산 (아이언4 0LP = 0pt, 디비전당 100pt, 마스터+ = 2800pt + LP)
4. 로비별 **절사평균**(표본 5명 이상이면 최고/최저 제외)으로 부캐·복귀 유저 이상치 완화
5. 오래된 경기부터 순차 추정: 로비 평균으로 관측 보정(35%) → **Elo 승패 업데이트**(K=32).
   강한 로비를 이기면 크게 오르고, 예상된 승리는 조금만 오르는 구조
6. 로비 평균들의 표준오차로 **95% 오차범위(±pt)** 를 함께 표기, 표본 수 기준 신뢰도 표시

### 정밀 분석 (표본 확대)

첫 검색은 빠른 추정(8경기 × 6명 표본)을 바로 보여주고, 백그라운드에서
**20경기 × 본인 제외 전원(9명)** 정밀 분석을 돌립니다. 진행률 배지가 표시되고
완료되면 페이지가 자동 갱신됩니다. 개발 키 기준 3~4분, Production 키면 수 초 소요.

분석 결과(빠른/정밀 모두)는 저장해 두고, 재검색 시 **최신 매치 ID가 그대로면
재분석 없이 그대로 재사용**합니다(매치 ID 1회 조회로 판단). 새 경기를 돌렸을
때만 다시 분석하며, 안전망으로 24시간 TTL을 둡니다.

- Production API Key(승인제, 무료)를 받으면 `.env.local`에 `RIOT_KEY_TYPE=prod` 추가
- 백그라운드 잡이 서버 메모리에 상주하므로 long-running 서버(`npm start`, Railway 등) 전제.
  Vercel 서버리스에서는 정밀 분석이 동작하지 않음

개발용 API 키의 호출 제한(20회/초, 100회/2분)에 맞춰 토큰 버킷 레이트리미터로
모든 호출을 직렬화하고, 매치 상세·랭크 조회 결과는 캐시에 저장해 재검색 시 호출을 아낍니다.

## Supabase 연결 (선택)

70명 규모면 무료 티어로 충분합니다.

1. https://supabase.com 에서 프로젝트 생성 (무료, 카드 불필요)
2. Project Settings → Database → Connection string (URI) 복사
3. `.env.local`에 `DATABASE_URL=...` 추가

첫 요청 시 `cache_entries` 테이블이 자동 생성됩니다. 별도 마이그레이션 불필요.
